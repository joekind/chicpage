2. 内容处理与转换引擎（关键实现）
这是该项目的技术核心，解决了从 Markdown 到“公众号兼容 HTML”的转换：

Markdown 解析：使用 marked 库将用户编写的 Markdown 实时解析为 HTML 结构。
CSS 内联化 (CSS Inlining)：为了绕过公众号编辑器对外部样式表的过滤，项目集成并深度使用了 juice。它能将用户自定义的 CSS 样式直接注入到 HTML 元素的 style 属性中。
DOM 修补与增强 (Hardening)：在复制前，代码对生成的 DOM 进行了细致的处理：
表格滚动优化：通过包装 section 并注入内联样式（overflow-x: auto），解决了移动端表格显示不全的问题。
微信白名单补丁：针对微信排版引擎的特性，对标题（H1-H6）进行了 Span 结构加固，并统一注入了兼容性的字体栈（如 Google Sans）。
3. 多端预览与模拟
设备仿真：集成了 Magic UI 的 Iphone 和 Safari 组件，利用 CSS 容器查询和缩放技术，让用户能直观看到内容在真机和 PC 浏览器上的实际效果。
4. 资源与资产管理
图片 Base64 处理：支持 拖拽上传 和 直接粘贴图片。实现逻辑会将图片转为 Base64 编码，并采用 Markdown 的**引用格式（Reference Style）**管理，既保证了实时预览，又方便了后续的整理。
本地持久化：使用 localStorage 动态保存用户的自定义模板、风格配置和图片资源，实现了简单的“离线编写”功能。
5. 交互与动效
Framer Motion：用于处理界面中的微交互和面板滑入滑出效果（如“CSS 实验室”侧边栏），提升了工具的使用手感。
